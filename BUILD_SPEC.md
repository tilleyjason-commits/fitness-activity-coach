# Fitness Activity Coach — Build Spec

## Overview
Build a mobile-first web app that tracks daily fitness/health activities and generates evidence-based recommendations. The app evaluates logged activities against ~40 coaching rules (distilled from peer-reviewed science and MASS Research Review) and surfaces actionable improvement suggestions.

## Stack
- React 18 + TypeScript + Vite
- Tailwind CSS 3 (mobile-first)
- Supabase (PostgreSQL, auth, real-time)
- React Router v6
- Chart.js or Recharts for weekly visualizations
- Deploy: Vite build → GH Pages (web) + Capacitor (mobile later)

## API Layer (Supabase Edge Functions OR Client-side)

Keep it client-side with Supabase JS client + Row Level Security. No edge functions needed for v1. The evaluation logic runs in-browser (load rules JSON + query daily_logs).

## Evaluation Engine (core logic)

File: `src/lib/evaluate.ts`

```typescript
// Load rules from /rules/rules.json
// For each rule whose trigger condition is met:
//   1. Check the evaluate expression against current day's daily_logs data
//   2. If failed → create recommendation entry
//   3. If passed and existing recommendation → dismiss it
// Run on: page load, after any log entry, and at midnight

// The evaluate field in rules is a JavaScript expression string
// that evaluates in a sandbox with the daily_log data as context.
// Use a simple Function() evaluator with safe access to the data:
function evaluateRule(rule, dailyLog) {
  const context = { ...dailyLog };
  try {
    const fn = new Function(...Object.keys(context), `return (${rule.evaluate})`);
    return fn(...Object.values(context));
  } catch (e) {
    console.error(`Rule ${rule.id} evaluation failed:`, e);
    return null;
  }
}
```

## UI / Design Guidelines

- **Mobile-first**: All screens work on a 390px wide phone screen
- **Dark mode as default** (with light toggle)
- **Color scheme**:
  - Background: slate-900 (#0f172a)
  - Cards: slate-800 (#1e293b)
  - Primary accent: emerald-500 (#10b981) — for pass/green
  - Danger: red-500 (#ef4444) — for fail/red
  - Warning: amber-500 (#f59e0b) — for medium issues
  - Info: blue-400 (#60a5fa) — for info
  - Text: slate-100 (#f1f5f9)
- **Typography**: Inter font family
- **Components**: Use shadcn/ui or custom minimal components (no heavy UI lib)
- **The gym cards** from the original program should appear as a "printable reference" section
